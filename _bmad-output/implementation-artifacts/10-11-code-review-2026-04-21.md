### HIGH-10.11-2 — `DATABASE_URL` absent → fallback silencieux `localhost:5432` + nightly break non détecté
**Source** : Edge Case Hunter E29, partiellement Blind Hunter F21.
**Fichiers** :
- `backend/app/core/config.py:35` : `database_url: str = "postgresql+asyncpg://postgres:postgres@localhost:5432/esg_mefali"`
- `.github/workflows/check-sources.yml:48` : `DATABASE_URL: ${{ secrets.STAGING_DATABASE_URL_READ_ONLY }}`
- `backend/scripts/check_source_urls.py:253-255` : `from app.core.config import settings` puis `await collect_urls_from_db(settings.database_url)`

**Scénario** :
1. Le secret `STAGING_DATABASE_URL_READ_ONLY` n'est pas provisionné (Task 11 explicite : « secret à provisionner avant le premier run nightly »).
2. Le workflow lance `python scripts/check_source_urls.py --output ../report.json` ; `settings.database_url` retombe sur le default `postgresql+asyncpg://postgres:postgres@localhost:5432/esg_mefali`.
3. `create_async_engine(...)` + `engine.connect()` → `ConnectError` (localhost:5432 inexistant sur le runner GitHub).
4. `asyncio.run(run(args))` propage l'exception → exit code ≠ 0 → `set -euo pipefail` fail le step AVANT que `echo has_failures=true >> $GITHUB_OUTPUT` ne soit exécuté.
5. Step « Create GitHub issue » est gardé par `if: steps.check.outputs.has_failures == 'true'` → **condition fausse**, pas d'issue créée.
6. L'alerting est cassé **silencieusement**. Personne ne sait que le nightly ne tourne plus.

**Combiné avec M10** (silent crash) : c'est le vrai risque opérationnel. Task 11 smoke est précisément faite pour détecter ça mais elle est **différée par design**.

**Correctif** :
```python
# backend/scripts/check_source_urls.py, après parse_args()
async def run(args: argparse.Namespace) -> int:
    if not args.dry_run:
        import os
        if not os.environ.get("DATABASE_URL"):
            logger.error(json.dumps({
                "level": "ERROR",
                "metric": "source_url_check",
                "error": "DATABASE_URL env var required (non-dry-run mode)"
            }))
            # Écrire un rapport minimal pour déclencher l'issue
            Path(args.output).write_text(json.dumps({
                "generated_at": _iso_utc(datetime.now(timezone.utc)),
                "total_sources_checked": 0,
                "counts": {k: 0 for k in _COUNT_KEYS},
                "sources": [{"source_url": "<boot>", "table": "boot",
                             "status": "other_error", "http_code": None,
                             "detected_at": _iso_utc(datetime.now(timezone.utc)),
                             "last_valid_at": None,
                             "suggested_action": "admin_check_mirror",
                             "duration_ms": 0}],
            }, indent=2), encoding="utf-8")
            return 0  # AC5 : toujours exit 0
    ...
```

Alternative plus simple : ajouter un step workflow `Pre-flight check` qui fail-fast si `secrets.STAGING_DATABASE_URL_READ_ONLY == ''` (via `if: env.DATABASE_URL == ''` + fallback issue).

### MEDIUM-10.11-2 — Détection SSL fragile par substring `"SSL"` dans le message d'exception
**Source** : Blind B7 + Edge E2.
**Fichier** : `backend/scripts/check_source_urls.py:133-134`

**Evidence** : `status = "ssl_error" if "SSL" in str(exc) else "other_error"`.

**Scénario** : httpx émet des `ConnectError` dont le texte varie (`[SSL: CERTIFICATE_VERIFY_FAILED]` OK, mais aussi `[X509]: unable to get local issuer certificate`, `certificate verify failed`, `TLS handshake failed`). Certains certs expirés réels finissent sans token `SSL` → catégorisé `other_error` → admin reçoit `admin_check_mirror` au lieu de `admin_verify_ssl`.

**Correctif** :
```python
except httpx.ConnectError as exc:
    import ssl
    cause = exc.__cause__
    if isinstance(cause, ssl.SSLError):
        status = "ssl_error"
    else:
        msg = str(exc).upper()
        status = "ssl_error" if any(k in msg for k in ("SSL", "TLS", "X509", "CERTIFICATE")) else "other_error"
```

### MEDIUM-10.11-3 — `asyncio.gather` sans borne de concurrence → rate-limit / ban IP à mesure que le catalogue croît
**Source** : Blind B8 + Edge E9.
**Fichier** : `backend/scripts/check_source_urls.py:266-267`

**Evidence** : `tasks = [check_one(...) for url, table in urls.items()] ; results = await asyncio.gather(*tasks)`.

**Scénario** : aujourd'hui 22 sources → gérable. Dès qu'admin N3 (Story 13.8) ajoute 50+ sources, gather lance tout en parallèle. Plusieurs URLs partagent le même host (`ifrs.org`, `sasb.ifrs.org` ; variations `boad.org`) → 429 / ban runner IP. Spec piège #2 « rate limiting 429 » annonce ce piège mitigé mais il ne l'est pas.

**Correctif** :
```python
limits = httpx.Limits(max_connections=10, max_keepalive_connections=5)
async with httpx.AsyncClient(..., limits=limits) as client:
    semaphore = asyncio.Semaphore(10)
    async def _guarded(url, table):
        async with semaphore:
            return await check_one(client, url, table)
    tasks = [_guarded(url, table) for url, table in urls.items()]
    results = await asyncio.gather(*tasks, return_exceptions=True)  # cf. M4
```

### MEDIUM-10.11-4 — `asyncio.gather` sans `return_exceptions=True` → une exception non-httpx tue tout le batch, AC5 violé
**Source** : Blind B9.
**Fichier** : `backend/scripts/check_source_urls.py:267`

**Scénario** : `check_one` catch 5 exceptions httpx explicites (bien, règle C1 9.7). Mais si une `KeyError` dans `_STATUS_TO_ACTION`, un `AttributeError`, un `TypeError` dans `_iso_utc` remonte, `gather` propage et abandonne **tous** les autres résultats. Le rapport n'est pas écrit. `run()` lève, `asyncio.run()` propage, `main()` retourne avec exception — le script **exit ≠ 0**, violation AC5 (exit 0 inconditionnel).

**Correctif** : `await asyncio.gather(*tasks, return_exceptions=True)` + filter `isinstance(r, Exception)` → log + coerce `other_error`.

### MEDIUM-10.11-5 — 429 (rate limit) catégorisé `other_error` sans respect `Retry-After` → piège #2 prétendument mitigé
**Source** : Edge E8.
**Fichier** : `backend/scripts/check_source_urls.py:87-100`

**Evidence** : `_categorize(429)` retombe sur `other_error` car 429 ∉ {200-299, 404, 500-599}. Le spec (piège #2) et CODEMAPS §2 annoncent « 429 → admin_check_mirror, un seul run/nuit ». ### MEDIUM-10.11-10 — `set -euo pipefail` + crash script avant écriture rapport → issue GH jamais créée (silent break)
**Source** : Edge E16.
**Fichier** : `.github/workflows/check-sources.yml:49-58`

**Lien avec HIGH-10.11-2**. Le script retourne toujours 0 **si** il atteint la fin (AC5). Mais une exception avant `return 0` (ex: `settings.database_url` lève une ValidationError Pydantic, ou `create_async_engine` crash sur import, ou `asyncio.run` propage une exception non catchée) → exit ≠ 0 → `set -e` tue le step → `echo has_failures=true` jamais exécuté → issue non créée. **Combine parfaitement avec H2** pour un nightly silencieusement cassé.

**Correctif** : wrapper `run()` dans un try/except au niveau de `main()` :
```python
def main() -> int:
    logging.basicConfig(level=logging.INFO, format="%(message)s")
    args = build_parser().parse_args()
    try:
        return asyncio.run(run(args))
    except Exception as exc:  # dernier recours, écrit un rapport minimal
        logger.exception("source_url_check crashed")
        Path(args.output).write_text(json.dumps({
            "generated_at": _iso_utc(datetime.now(timezone.utc)),
            "total_sources_checked": 1,
            "counts": {k: (1 if k == "other_error" else 0) for k in _COUNT_KEYS},
            "sources": [{"source_url": "<script_crash>", "table": "runtime",
                         "status": "other_error", "http_code": None,
                         "detected_at": _iso_utc(datetime.now(timezone.utc)),
                         "last_valid_at": None,
                         "suggested_action": "admin_check_mirror",
                         "duration_ms": 0}],
        }, indent=2), encoding="utf-8")
        return 0  # AC5 strict
```

### MEDIUM-10.11-11 — Markdown injection dans `format_issue_body.py` via URLs admin N3
**Source** : Blind B12.
**Fichier** : `backend/scripts/format_issue_body.py`

**Scénario** : les URLs non-seed proviennent de `sources.url` (N3 admin, Epic 13.8). Une URL contenant `|` ou `` ` `` casse le tableau Markdown ou sort d'une code span, autorisant HTML inline dans le render GitHub. Un admin malveillant ne peut pas grand-chose (repo privé), mais un admin inattentif peut casser le rendu issue.

**Correctif** : échapper les cellules avant insertion :


### MEDIUM-10.11-13 — GET fallback sans `stream=True` → bande passante et timeout si serveur ignore `Range`
**Source** : Edge E5.
**Fichier** : `backend/scripts/check_source_urls.py:124-126`

**Scénario** : `response = await client.get(url, headers={"Range": "bytes=0-1023"})` — httpx buffer le body complet par défaut. Si le serveur ignore `Range` et renvoie 200 + PDF complet (30 MB BOAD SSI), on télécharge tout. Timeout par défaut 10 s → majeure partie en `timeout` si réseau lent.
