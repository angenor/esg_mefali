### [MEDIUM-10.4-1] Test 17 ne vérifie pas le compteur `len(INSTRUMENTED_TOOLS) == 36` exigé par AC8

**Fichier** : `backend/tests/test_graph/test_admin_catalogue_absence_from_graph.py:46-60`

**Constat** : l'AC8 #17 stipule explicitement :

> `test_no_admin_catalogue_tool_in_instrumented_tools` → `INSTRUMENTED_TOOLS count inchangé (=36 post-10.3)` **ET** aucun tool avec `node_name in {"admin", "admin_catalogue", "catalogue"}` (AC6)

Le test livré ne vérifie **que la 2ᵉ assertion** :

```python
def test_no_admin_catalogue_tool_in_instrumented_tools():
    from app.graph.tools import INSTRUMENTED_TOOLS

    # Le count exact peut varier au cours des Epic — on verifie surtout
    # qu'aucun tool admin_catalogue n'est present par son nom.
    for tool in INSTRUMENTED_TOOLS:
        name = getattr(tool, "name", "")
        assert not name.startswith("admin_"), ...
        assert "catalogue" not in name.lower(), ...
```

Le commentaire « le count exact peut varier au cours des Epic » **contredit** l'AC8 qui exige `count=36` comme invariant figé post-10.4. Rendre le count « variable » n'est pas un compromis valable : si Epic 11 ajoute un tool `projects_tools` sans toucher à `admin_catalogue`, le compteur passera à 37 — ce **n'est pas** une régression admin_catalogue et le test doit continuer de passer. Mais si quelqu'un ajoute en catimini un tool dans `admin_catalogue_tools.py` (fichier déjà interdit par test 16), le compteur franchirait 37 sans détection **dans ce test** — le test 16 (`admin*.py` scan) attrape déjà ce cas, donc le filet est redondant mais l'invariant AC8 reste littéralement non testé.

**Risque** : faible opérationnel (test 16 capture 95 % du risque), mais l'AC n'est pas littéralement satisfait → **gap de traçabilité**.

**Recommandation (choisir 1)** :

- **Option A (recommandée, 2 lignes)** — ancrer la valeur attendue dans un constant local et tolérer l'augmentation via Epic 11/12/13 lorsque ces stories livreront leurs tools :

  ```python
  def test_no_admin_catalogue_tool_in_instrumented_tools():
      from app.graph.tools import INSTRUMENTED_TOOLS

      # AC8 #17 — count figé post-10.4. Mise à jour requise lors de futures Stories ajoutant des tools.
      assert len(INSTRUMENTED_TOOLS) == 36, (
          f"INSTRUMENTED_TOOLS count a changé ({len(INSTRUMENTED_TOOLS)} vs 36 attendu post-10.4). "
          f"Si intentionnel (nouvelle story avec tool), mettre à jour cette valeur. "
          f"Si admin_catalogue a ajouté un tool, REJETER (module UI-only)."
      )
      for tool in INSTRUMENTED_TOOLS:
          ...
  ```

- **Option B** — déférer et documenter l'intention de relâcher AC8 vers test par noms uniquement (explicite dans `deferred-work.md`). Cohérent avec la réalité mouvante Epic 11/12/13 mais requiert amendement d'AC8 #17.

**Décision souhaitée** : `A` (2 lignes, ferme l'AC littéralement, message d'erreur pédagogique) OU `B` (amendement AC8 explicite).
