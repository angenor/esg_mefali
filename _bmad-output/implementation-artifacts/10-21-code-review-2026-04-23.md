## FINDINGS DÉTAILLÉS

### HIGH — 1

#### **H-1** `EsgIcon.vue:163` — Warn dev-only non-réactif (`name` prop dynamique silencieuse)

**Source** : Blind Hunter HIGH + Edge Case Hunter MEDIUM (consolidé HIGH par convergence)

```ts
if (import.meta.env.DEV && !(props.name in ICON_MAP)) {
  console.warn(`[EsgIcon] Unknown icon name: "${props.name}". ...`);
}
```

Bloc exécuté **une seule fois au setup**, hors contexte réactif. Si `:name` est lié à une variable réactive (`v-for`, registres dynamiques) et passe de `'check'` à `'unknown-xyz'` après mount, le warn ne se redéclenche pas — le placeholder s'affiche silencieusement. Test `warnSpy` ne couvre pas ce cas (composant monté une fois avec name invalide).

**Fix (5 min)** : encapsuler dans `watchEffect` ou fusionner dans `resolvedComponent` :
```ts
const resolvedComponent = computed<Component>(() => {
  const resolved = ICON_MAP[props.name];
  if (!resolved && import.meta.env.DEV) {
    console.warn(`[EsgIcon] Unknown icon name: "${props.name}". Falling back to placeholder.`);
  }
  return resolved ?? PlaceholderIcon;
});
```

#### **M-2** `EsgIcon.vue:196` — `:size` forwarded aux SVG custom (attribut HTML non-conforme)

**Source** : Edge Case Hunter

`<component :is="resolvedComponent" :size="pixelSize">` passe `:size` à TOUS les composants résolus, y compris les 6 SVG custom chargés via `vite-svg-loader?component`. Ces composants ne déclarent pas de prop `size` → Vue forwarde comme attribut HTML brut `size="20"` sur l'élément `<svg>` racine — **attribut HTML non-standard sur SVG**. Le browser l'ignore silencieusement, mais le DOM est non-conforme et le sizing diverge entre Lucide (prop native size) et custom (sizing CSS uniquement).

**Fix (15 min)** : conditionner le forward avec une `Set` des noms custom OU ajouter `:style="{ width: pixelSize+'px', height: pixelSize+'px' }"` uniformément (override prop).

---

#### **M-3** `EsgIcon.vue` — `inheritAttrs` non désactivé : conflit ARIA silencieux possible

**Source** : Edge Case Hunter

Pas de `defineOptions({ inheritAttrs: false })`. Si un consommateur passe `<EsgIcon name="x" decorative aria-label="fermer" />`, le DOM produit `aria-hidden="true" aria-label="fermer"` — combinaison **interdite par ARIA spec** (axe/pa11y la signaleront). Pas un bug actuel (aucun consommateur connu ne fait ça), mais wrapper ARIA sans `inheritAttrs: false` = défense en profondeur manquante.

**Fix (10 min)** : `defineOptions({ inheritAttrs: false })` + `v-bind="$attrs"` filtré sur les clés ARIA.

## LEÇONS §4OCTIES CANDIDATES (généralisables Epic 11+)

### **L31 — CI gate tree-shaking sur dépendances catalogue**

L28 (named imports stricts) est documentée mais enforcement manuel. Pattern : ajouter step CI `rg "import \* as.*(lucide|lodash|date-fns|heroicons|react-icons)" frontend/app --count` → fail si > 0. Coût 10 min, prévient régression silencieuse à chaque ajout de lib catalogue. **Issu Acceptance §4octies-1.**

### **L32 — Traçabilité DEF cross-story (closure obligatoire)**

Un DEF qui pointe vers une story future doit produire à la résolution : (a) fermeture explicite "RÉSOLU + commit ref", (b) re-déférement formel `DEF-X.Y-N` avec rationale, ou (c) retrait obsolète. DEF-10.15-1 illustre la dette : pointait vers 10.21, traité par re-justification implicite sans tracker successeur. **Issu Acceptance §4octies-2.**

### **L33 — Baseline canonique unique commit↔Debug Log**

Métriques Pattern B numériques doivent être byte-identiques entre commit message et Debug Log. Pre-commit hook `grep -E "[0-9]+ → [0-9]+" commit-msg` vs story file. Coût nul, élimine divergences audit type "832/821". **Issu Acceptance §4octies-3.**

### **L34 — Warn dev-only doit être réactif (NOUVELLE — issue cette review)**

Tout `console.warn` conditionné à `import.meta.env.DEV` sur prop dynamique doit vivre dans `watchEffect` ou `computed`, **jamais au top-level `<script setup>`**. Sinon : warn manqué sur prop changeante après mount, faux sentiment de sécurité côté tests (qui montent une fois). Pattern généralisable à tous fallback warnings (registry lookups, schema mismatches, mode flags). **Issu convergence Blind+Edge Case sur H-1.**

### **L35 — `inheritAttrs: false` obligatoire pour wrappers ARIA (NOUVELLE — issue cette review)**

Tout composant qui définit ses propres attrs ARIA via `v-bind="ariaAttrs"` doit `defineOptions({ inheritAttrs: false })` pour empêcher les conflits silencieux fallthrough (ex: `aria-hidden + aria-label` simultanés). Sinon a11y défaut potentiel sur consommateur incorrect. Pattern généralisable à tous wrappers a11y (Modal, Dialog, Tooltip, MenuItem). **Issu Edge Case M-3.**
