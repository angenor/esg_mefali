### [MEDIUM-10.3-1] `test_country_data_driven_no_hardcoded_country_strings` ne couvre que 2 fichiers du module sur 8

**Fichier** : `backend/tests/test_maturity/test_formalization_plan_calculator.py:53-56`

**Constat** : la liste `targets` du scan NFR66 est limitée à :

```python
targets = [
    _MATURITY_MODULE_DIR / "service.py",
    _MATURITY_MODULE_DIR / "formalization_plan_calculator.py",
]
```

Le module `backend/app/modules/maturity/` contient **8 fichiers Python** (`__init__.py`, `enums.py`, `events.py`, `models.py`, `schemas.py`, `router.py`, `service.py`, `formalization_plan_calculator.py`). Epic 12 pourrait introduire un hardcoding pays dans, par exemple :
- `schemas.py` via un `Field(examples=["Sénégal"])` ou un `Literal["Sénégal", "Côte d'Ivoire"]`.
- `enums.py` via une éventuelle `CountryEnum` (anti-pattern NFR66, mais que seul le scan empêcherait).
- `router.py` via un docstring OpenAPI avec exemple de pays.
- `events.py` ou `models.py` via une docstring.

Le scan actuel **raterait** ces cas. Or, le rôle même de la défense NFR66 — telle que posée par la story (§« Country-data-driven NFR66 — LA DETTE QUE CETTE STORY PRÉVIENT ») — est de bloquer l'ensemble du module, pas seulement 2 fichiers.

**Risque** : Epic 12.3 `FormalizationPlanCalculator.generate` est la principale source de dette NFR66 potentielle, **mais** Epic 12.1 (router + schemas validation) est tout aussi vulnérable à des exemples Pydantic figeant des pays. Le test passerait green et la dette se glisserait silencieusement.

**Recommandation** (choisir 1) :

- **Option A (recommandée)** : étendre le scan à tout `*.py` du module, en itérant sur `_MATURITY_MODULE_DIR.glob("*.py")` (exclure `__pycache__`) :

  ```python
  targets = sorted(p for p in _MATURITY_MODULE_DIR.glob("*.py") if p.name != "__init__.py")
  # ou, si __init__.py doit aussi être scanné (recommandé) :
  targets = sorted(_MATURITY_MODULE_DIR.glob("*.py"))
  ```

  Puis message d'erreur enrichi pour dire quel fichier contient la string. Coût : 3 lignes.

- **Option B** : dupliquer le scan dans un second test `test_country_data_driven_scans_full_module` qui itère sur tous les fichiers, en gardant le test existant pour `service.py` + `formalization_plan_calculator.py` (plus verbeux mais explicite).

- **Option C** : transformer le test en test global `tests/test_architecture/test_nfr66_no_hardcoded_countries.py` scannant **tout `backend/app/modules/maturity/`** ET `backend/app/graph/tools/maturity_tools.py` (le tool peut aussi introduire un hardcoding). Capitalise le pattern pour les futures stories country-sensitive (Epic 12).

**Décision souhaitée** : `A` (coût minimal, scope strict story 10.3) OU `C` (investissement défense future, 15 min).
